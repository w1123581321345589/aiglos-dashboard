March 2026 CVE Disclosure

Two new CVEs: supply chain RCE and A2A spoofing

Aiglos researchers identified two new vulnerabilities in the MCP ecosystem during routine fuzzing of the T30 Supply Chain and T25 A2A Protocol rule families.

CVE-2026-26319
CVSS 9.0 — Critical
Remote code execution via malicious tool registration overriding a trusted server namespace. An attacker controlling a malicious MCP server can register a tool with the same name as a trusted tool, causing the client to execute attacker-controlled code.
● In coordinated disclosure
CVE-2026-26322
CVSS 7.7 — High
Agent delegation spoofing via crafted A2A task message. A malicious agent can forge a task message appearing to originate from a trusted orchestrator, causing unauthorized tool execution under a different agent's permissions.
● In coordinated disclosure

Detection rules for both are included in aiglos-core v1.2.0 and above. Patches are in coordinated disclosure with affected vendors.

February 2026 CVE Disclosure Security

CVE-2026-25253: WebSocket localhost hijack — CVSS 8.8

A malicious MCP server can register a tool that proxies calls to a locally running WebSocket server, bypassing same-origin restrictions and gaining access to local-only services including Claude Desktop's internal API surface.

CVE-2026-25253
CVSS 8.8 — Critical
WebSocket localhost hijack via malicious MCP tool registration. Affected: any MCP client running on a host with locally bound WebSocket services. Patch: aiglos-core v1.1.3 adds T35 WebSocket guard blocking ws://localhost:* and ws://127.0.0.1:* destinations in tool arguments.
✓ Patched in v1.1.3

January 2026 CVE Disclosure Security

CVE-2026-25593: SSRF via MCP file-fetch tool

A tool argument validation gap allows agents to reach internal network endpoints via a seemingly innocuous file-fetch MCP tool. The vulnerability was discovered during T13 Network rule family development.

CVE-2026-25593
CVSS 5.9 — Medium
SSRF via internal network probe through file-fetch MCP tool. The tool accepts a URL argument with no scheme or host restrictions, allowing requests to http://169.254.169.254 (AWS metadata), http://10.x.x.x (internal networks), and file:// URIs.
✓ Patched in v1.0.4
